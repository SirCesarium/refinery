# Packaging System

Refinery distinguishes between technical compilation outputs and final distribution packages to provide a clean and professional delivery workflow.

## Build Types vs. Packages

### Build Types (`library_types`)
These are internal formats generated by the compiler.
- **`bin`**: A standalone executable.
- **`cdylib`**: A C-compatible dynamic library (`.so`, `.dll`, `.dylib`).
- **`staticlib`**: A static library (`.a`, `.lib`).

### Packages (`packages`)
These are the files intended for the end-user or for system integration.
- **`tar.gz` / `zip`**: Portable archives.
- **`deb`**: Debian/Ubuntu installer.
- **`rpm`**: RedHat/CentOS/Fedora installer.
- **`msi`**: Windows installer.

## SDK Bundling (Archives)

When a `lib` type artifact specifies an archive format (`zip` or `tar.gz`) in the `packages` list, Refinery performs **SDK Bundling**:

1.  **Multi-Format Aggregation**: It collects all variants defined in `library_types` (e.g., both the static and dynamic versions of the library) and places them in the same archive.
2.  **Header Inclusion**: If `headers = true` is set, Refinery scans the project root for `.h` and `.hpp` files and includes them in the archive.
3.  **Result**: The user receives a single file containing everything needed to develop against your library.

## System Packages (Native Installers)

For native packages like `.deb` or `.msi`, Refinery orchestrates language-specific tools to ensure all metadata (description, author, license) is correctly embedded.

### Rust Integration
- **`deb`**: Orchestrates `cargo-deb`.
- **`rpm`**: Orchestrates `cargo-generate-rpm`.
- **`msi`**: Orchestrates `cargo-wix`.

Refinery manages the installation of these tools in CI environments automatically, ensuring that the packaging phase succeeds without manual runner configuration.

## Output Organization

All generated packages are organized in the `output_dir` (default `dist/`) according to the templates defined in the `[naming]` section. This ensures a consistent and predictable structure for CI artifact uploads and release publishing.
